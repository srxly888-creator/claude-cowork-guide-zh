# 上下文文件模板

> **复制到你的 00_Context/ 文件夹**

---

## about-me.md
```markdown
# 关于我

## 角色 & 职责
- [你的名字、 职位、 公司]
- [你的日常工作内容]
- [关键利益相关者]
- [成功的定义]

## 领域背景
- [行业/领域细节]
- [关键术语或 框架]
- [工作流程中的工具和平台]

## 示例工作
[粘贴 1-2 个你引以为豪的输出示例 — 报告、邮件、 分析。
这给 Claude 具体的质量和风格参考。]
```

---

## brand-voice.md
```markdown
# 沟通风格

## 语调
- [例如： 直接简洁。 无废话。 适时专业。]
- [你自然使用的短语]
- [你觉得不对的短语]

## 写作示例
[粘贴 2-3 个你的实际写作示例 — 鸿件、 帖子、 文档。
这些比抽象描述更有用。]

## 反模式
- [例如： 永远不要把 "leverage" 当动词用]
- [例如： 不要用 "希望你收到这封邮件时一切顺利" 开头]
```

---

## working-preferences.md
```markdown
# 我希望 Claude 如何工作

## 流程
- 在开始非平凡任务前始终询问澄清问题
- 执行前向我展示计划
- 输出保存为 [.docx / .xlsx / .md — 你的偏好]

## 输出风格
- [简短 vs 详细输出]
- [首选格式约定]
- [文件命名约定]

## 安全边界
- 没有明确确认， 不要删除文件
- 不要修改指定输出文件夹之外的文件
- 行动前明确标记假设
```
